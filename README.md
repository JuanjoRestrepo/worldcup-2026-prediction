# World Cup 2026 Prediction Pipeline

End-to-end system for ingesting international football data, engineering robust match features, training a reproducible classifier, curating medallion layers in PostgreSQL/dbt, and serving predictions through FastAPI.

## Current Stack

- Python for ingestion, processing, modeling, and API logic
- PostgreSQL in Docker for persisted bronze/silver/gold and monitoring tables
- dbt for curated serving/monitoring models plus data quality checks
- FastAPI for prediction and lightweight monitoring endpoints
- `uv` for local Python environment and command execution

## Setup

1. Copy `.env.example` to `.env`
2. Start PostgreSQL:

```bash
docker compose up -d postgres
```

3. Sync the local Python environment with `uv`:

```bash
uv sync
```

## Data Layers

- `data/raw`: immutable CSV source plus raw API payloads
- `data/bronze`: standardized extracts generated by ingestion
- `data/silver`: cleaned canonical match-level table
- `data/gold`: model-ready feature dataset

## Pipeline Flow

1. Ingestion writes raw snapshots and standardized bronze data
2. Processing writes `data/silver/matches_cleaned.csv`
3. Processing writes `data/gold/features_dataset.csv`
4. Training exports `models/match_predictor.joblib`
5. Optional persistence writes `bronze.*`, `silver.*`, `gold.*` to PostgreSQL
6. dbt curates serving and monitoring models from persisted layers
7. FastAPI serves predictions and logs each inference to `monitoring.inference_logs`

## Run The Pipeline

```bash
uv run python run_pipeline.py
```

Useful variants:

```bash
uv run python run_pipeline.py --skip-ingestion
uv run python run_pipeline.py --skip-ingestion --skip-processing
uv run python run_pipeline.py --no-api-data
uv run python run_pipeline.py --persist-to-db
```

When `--persist-to-db` is enabled, the pipeline writes:

- `bronze.historical_matches`
- `bronze.api_matches`
- `silver.matches_cleaned`
- `gold.features_dataset`
- `gold.training_runs`

## Run Training

```bash
uv run python -m src.modeling.train
```

This now performs:

- rolling temporal backtesting across candidate models
- post-selection calibration checks with `uncalibrated`, `sigmoid`, and `isotonic` variants
- deployment gating so calibration is only exported when it improves probability quality without materially hurting macro-F1 or draw recall

The command exports:

- `models/match_predictor.joblib`
- `models/match_predictor_metrics.json`

## Run dbt

Recommended sequence after a persisted pipeline run:

```bash
uv run python run_dbt.py debug
uv run python run_dbt.py source freshness
uv run python run_dbt.py run
uv run python run_dbt.py test
```

Serving prefers dbt-curated models when available:

- `analytics_gold.gold_latest_team_snapshots`
- `analytics_gold.gold_latest_training_run`

Historical serving for `match_date` requests uses:

- `analytics_gold.gold_team_feature_snapshots`

## Run The API

```bash
uv run uvicorn src.api.main:app --host 0.0.0.0 --port 8000 --reload
```

Interactive docs:

- `http://localhost:8000/docs`

Key endpoints:

- `POST /predict`
- `GET /monitoring/latest-training-run`
- `GET /monitoring/inference-stats`
- `GET /monitoring/recent-inferences`
- `GET /config`

Prediction serving uses `PREDICTION_FEATURE_SOURCE=auto` by default:

- latest predictions prefer `dbt` latest team snapshots
- historical predictions with `match_date` prefer `dbt` team snapshots as of that date
- fallback path uses persisted `gold.features_dataset`
- final fallback uses local `data/gold/features_dataset.csv`

Monitoring uses `MONITORING_SOURCE=auto` by default:

- prefers `analytics_gold.gold_latest_training_run`
- falls back to raw `gold.training_runs`

Example prediction request:

```bash
curl -X POST "http://localhost:8000/predict" ^
  -H "Content-Type: application/json" ^
  -d "{\"home_team\":\"USA\",\"away_team\":\"Argentina\",\"tournament\":\"FIFA World Cup Qualifiers\",\"neutral\":false,\"match_date\":\"2025-11-18\"}"
```

Example monitoring requests:

```bash
curl "http://localhost:8000/monitoring/latest-training-run"
curl "http://localhost:8000/monitoring/inference-stats?hours=24"
curl "http://localhost:8000/monitoring/recent-inferences?limit=20"
```
